pylmgc90.chipy.macro.ExSolver(stype, norm, conv, relax, gsit1, gsit2)[source]¶

Run Non-linear Gauss Seidel contact solver.

parameters:

  • stype: type of solver (‘Exchange Local Global’ or ‘Stored_Delassus_Loops’).

  • norm: type of norm (‘Quad’, ‘QM/16’, ‘Maxm’).

  • conv: convergence tolerance demanded.

  • relax: relaxation parameter.

  • gsit1: maximum number of convergence check before stopping.

  • gsit2: number of iterations of nlgs before checking convergence.

pylmgc90.chipy.macro.SelectProxTactors(freq_detec=1, useExt=0, reset=0)[source]¶

Run detections if current time step number is a multiple of freq_detec.

In case of PRPRx contact, one of next functions must be called:

  • PRPRx_UseCpCundallDetection

  • PRPRx_UseCpF2fExplicitDetection

  • PRPRx_UseCpF2fDetection

  • PRPRx_UseNcDetection

  • PRPRx_UseNcF2fDetection

  • PRPRx_UseNcF2fExplicitDetection

Parameters:

  • freq_detec (default 1) : frequency of particules neighbourhood recomputation during detection

  • useExt (default 0) : set to 1 to use external detections of CLALp interactions (coupling with Xper)

  • reset (default 0) : set to 1 to reset the detection and redo box computation

pylmgc90.chipy.computation.initialize(dim, dt, theta, mhyp=0, h5_file=None, deformable=False, logmes=False, restart=None)[source]¶

Initialize a (linear) LMGC90 simulation with some default parameters.

Parameters

  • dim – (integer) dimension of the simulation (2 or 3)

  • dt – (real) time step of the simulation

  • theta – (real) value of the theta integrator ( value in [0.,1.])

  • mhyp – (integer) modeling hypothesis use for deformable bodies: * 0 = 3D (default value) * 1 = plain strain * 2 = plain stress * 3 = axi-symmetry

  • h5_file – (string optional) HDF5 file in which to save the computation. If not set, only text files in the OUTBOX directory will be available.

  • deformable – (boolean) with deformable bodies (default to false)

  • logmes – (boolean optional) set to True to activate LMGC90 log messaging.

  • restart – (one or two integers) restart parameters to put in ReadIni and OpenDisplayFiles

pylmgc90.chipy.computation.initialize_non_linear(dim, dt, theta, mhyp, t_final, dt_min, dt_max, NR_max_iter, NR_adapt, h5_file=None, logmes=False)[source]¶

Initialize an LMGC90 simulation involving non linear deformable bodies

Parameters

  • dim – (integer) dimension of the simulation (2 or 3)

  • dt – (real) time step of the simulation

  • theta – (real) value of the theta integrator ( value in [0.,1.])

  • mhyp – (integer) modeling hypothesis use for deformable bodies: * 0 = 3D * 1 = plain strain * 2 = plain stress * 3 = axi-symmetry

  • t_final – (real) desired final time of the simulation

  • dt_min – (real) mininum time step allowed in the adaptative process

  • dt_max – (real) maxinum time step allowed in the adaptative process

  • NR_max_iter – (integer) maxinum number of iteration allowed in the Newton-Raphson loop

  • NR_adapt – (integer) number of consecutive iterations with same status allowing to decide if the time step should increase of decreased.

  • h5_file – (optional) HDF5 file in which to save the computation. If not set, only text files in the OUTBOX directory will be available.

  • logmes – (optional) set to True to activate LMGC90 log messaging.

pylmgc90.chipy.computation.one_step(stype, norm, tol, relax, gs_it1, gs_it2, f_write, f_display)[source]¶

Compute one step of a computation with rigids or linear deformable bodies.

Parameters

  • stype – type of contact solver to use can only be: * ‘Stored_Delassus_Loops ‘ * ‘Exchange Local Global ‘

  • norm – type of norm to use in contact solver to check convergence, can be: * ‘Quad ‘ * ‘Maxm ‘ * ‘QM/16’

  • tol – (real) desired tolerance to decided if contact solver has converged.

  • relax – (real) relaxation

  • gs_it1 – (integer) maximum number of converge check of the contact solver before stopping (outer loop).

  • gs_it2 – (integer) number of contact solver iteration to run before checking convergence (inner loop).

  • f_write – (integer) frequency at which to save into file(s).

  • f_display – (integer) frequency at which to save display files (if 0, no file generated).
